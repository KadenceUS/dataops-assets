Schedule Summary Methods¶

Clean Data¶

Clean up a dataframe according to user parameters.

Parameters:
  • data_source (string) –

    String that denotes where to pull data from. Extract.Data Entry would look for any Data Entry sheets made in the extract step. 'Transformations.2` would mean to take the results from the second transformation step.

  • blacklist_filter (dict, default: None ) –

    Dictionary of column and blacklisted entries to filter out. Defaults to None.

  • mappings (dict, default: None ) –

    Dictionary of columns and mappings (dict format) to perform on the series. Defaults to None.

  • drop_duplicate_cols (list / string, default: None ) –

    A string or list of strings of columns to use for subset dropping of duplicates. Defaults to None.

 
Aliases: clean_data


Merge Calendar Data¶

Merge calendar data into another data source.

Parameters:
  • data_source (string) –

    String that denotes where to pull data from. Extract.Data Entry would look for any Data Entry sheets made in the extract step. 'Transformations.2` would mean to take the results from the second transformation step.

  • accumulator_source (string) –

    String that denotes where to pull data from (this is the dataframe that will have calendar data added to it) Extract.Data Entry would look for any Data Entry sheets made in the extract step. 'Transformations.2` would mean to take the results from the second transformation step.

  • var_name (str, default: 'Date' ) –

    New column name made from the melted calendar. Defaults to 'Date'.

  • value_name (str, default: 'Unique ID' ) –

    Value name argument to pass to PD.melt. Defaults to 'Unique ID'.
Raises:
  • TypeError –

    If the user has references something that is neither a list of dataframes, or a dataframe, raise an error.

 
Aliases: add_calander_data


Apply Windowing to Date Data¶

Make a windowed view of data using a window size in number of days.

Parameters:
  • data_source (string) –

    String that denotes where to pull data from. Extract.Data Entry would look for any Data Entry sheets made in the extract step. 'Transformations.2` would mean to take the results from the second transformation step.

  • window_size (int, default: 5 ) –

    Number of days to use for windowing. Defaults to 5.

 
Aliases: window_ScheduleSummary
